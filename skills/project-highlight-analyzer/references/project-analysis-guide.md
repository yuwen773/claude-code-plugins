### **一、文档目的**
本指南专为指导AI（大模型，如Grok、GPT等）而设计。  
当用户提供项目代码仓库（GitHub链接、ZIP包、关键文件描述或目录结构）时，AI需按照本指南**先结构化分析项目代码结构与技术栈**，再**严格结合《2025-2026 Java后端出色简历指导》**，最终输出**可直接复制粘贴到简历**的项目经验、技术栈亮点、自我评价等产出。  

目标：让简历中的项目描述**STAR结构化 + 量化 + 前沿 + 可追问**，充分体现2026年Java后端核心竞争力（Java 25、Spring Boot 4.x、Spring AI 2.0+、GraalVM、云原生、AI工程化、国产化等）。
